## ⚠️ Pontos importantes (pra não fazer errado)

* Pró-labore **não é distribuição de lucro** → é despesa operacional
* INSS entra como **despesa + passivo**, não direto no banco
* DAS **não reduz receita**, é despesa tributária
* Distribuição de lucro (se fizer) entra separado depois

1 Fncionário - Secretária

Vamos direto ao ponto — para CLT você precisa separar **3 coisas diferentes** que muita gente mistura:

1. **Despesa da empresa (salário + encargos patronais)**
2. **Descontos do funcionário (INSS, FGTS não é desconto)**
3. **Obrigações a pagar (INSS, FGTS, salário líquido)**

Vou montar um exemplo completo em **hledger** com salário de **R$ 1.600**.

# 📌 Premissas (simplificadas e realistas)

* Salário bruto: **1.600**

* INSS empregado (~8%): **128,00**

* FGTS (8%): **128,00**

* INSS patronal (20%) → ⚠️ depende do Simples
  👉 vou assumir **empresa fora do CPP do Simples**: **320,00**

* Salário líquido:
  👉 **1.600 - 128 = 1.472**

```txt
    liabilities:salarios_a_pagar  -1472.00 BRL

    liabilities:fgts_a_pagar      -128.00 BRL
```

* Empresa gasta:

  * salário = 1600
  * INSS patronal = 320
  * FGTS = 128

  * INSS = 128 (vira dívida da empresa com o governo)

* Empresa fica devendo:

  * líquido pro funcionário
  * INSS total (retido + patronal)
  * FGTS

# 📌 O que significa “empresa fora do CPP do Simples”

👉 Significa:

> A empresa **NÃO paga o INSS patronal dentro do DAS**,
> tendo que calcular e pagar **separadamente sobre a folha** (como no lucro presumido).

---

# 🧾 Primeiro: o que é CPP?

CPP = **Contribuição Previdenciária Patronal**
👉 basicamente o **INSS da empresa**

* É a parte que o **empregador paga**, diferente do INSS descontado do funcionário ([Cora][1])
* Normalmente é **20% sobre a folha de pagamento** ([CRCSP][2])
* Serve para financiar a Previdência Social ([Contabilizei][3])

---

# 🧠 Agora vem o ponto chave (Simples Nacional)

No Simples existem **2 cenários completamente diferentes**:

---

## ✅ 1. Empresa “DENTRO do CPP” (mais comum)

👉 CPP já está embutido no DAS

* Não calcula INSS patronal separado
* Não lança 20% sobre folha
* Não paga GPS/DARF de INSS patronal

✔ Isso acontece nos anexos:

* Anexo I
* Anexo II
* Anexo III
* Anexo V ([Cora][1])

👉 Aqui a CPP já está “escondida” dentro da alíquota do Simples

---

## ❌ 2. Empresa “FORA do CPP” (o que você perguntou)

👉 CPP NÃO está no DAS

* Precisa calcular:

  * **20% sobre a folha**
* Precisa pagar separado:

  * via **INSS (DCTFWeb/DARF)**

✔ Isso acontece no:

### 🔴 Anexo IV

Exemplos típicos:

* construção civil
* limpeza
* vigilância
* advocacia ([Contabilizei][4])

👉 Nesses casos:

> “A CPP não está incluída na alíquota do Simples Nacional” ([Serviços e Informações do Brasil][5])

---

# 📊 Tradução prática (pra você usar no dia a dia)

## 💡 Quando você vê:

👉 “empresa fora do CPP do Simples”

Significa:

✔ Tem que lançar no hledger:

```txt
expenses:inss_patronal
liabilities:inss_a_pagar
```

✔ Tem custo real de folha maior
✔ Impacta fortemente o caixa

---

## 💡 Quando está “dentro do CPP”

👉 NÃO lança INSS patronal separado

✔ Só:

* salário
* FGTS
* INSS retido

---

# ⚠️ Aplicando isso para médicos PJ

Aqui está o detalhe que separa contador comum de especialista:

### 👨‍⚕️ Médicos normalmente:

* começam no **Anexo V**
* podem ir para **Anexo III (fator R)**

👉 Em ambos os casos:
✅ CPP está **dentro do DAS**

✔ Ou seja:

> **NÃO existe INSS patronal de 20% na folha**

---

# 🚨 Erro clássico (e caro)

Muitos contadores fazem isso errado:

* lançam INSS patronal de 20%
* mesmo estando no Anexo III/V

👉 Isso **superestima custo** e distorce DRE

---

# 🧠 Resumo definitivo

| Situação            | CPP                           |
| ------------------- | ----------------------------- |
| Anexo I, II, III, V | ✔ dentro do DAS               |
| Anexo IV            | ❌ fora do DAS (paga separado) |


Isso aqui vira um diferencial absurdo no seu escritório.

# 🔧 4. Como extrair isso do hledger (automático)

Você pode puxar os dados assim:

```bash
hledger register expenses:salarios -b 2026-05 -e 2026-06
```

## 👉 INSS retido

```bash
hledger register liabilities:inss_retido -b 2026-05 -e 2026-06
```

## 👉 FGTS

```bash
hledger register expenses:fgts -b 2026-05 -e 2026-06
```

## 👉 Pró-labore

```bash
hledger register expenses:prolabore -b 2026-05 -e 2026-06
```

# 🤖 5. Automação real (nível escritório contábil)

Aqui está o caminho que recomendo pra você:

## Opção A – Script Python simples

* lê saída do hledger (CSV)
* calcula:

  * bruto
  * descontos
  * líquido
* gera PDF

### Exemplo conceitual:

```python
import subprocess

def get_value(account):
    result = subprocess.run(
        ["hledger", "balance", account, "-O", "csv"],
        capture_output=True, text=True
    )
    return float(result.stdout.splitlines()[-1].split(",")[-1])

salario = get_value("expenses:salarios")
inss = abs(get_value("liabilities:inss_retido"))
fgts = get_value("expenses:fgts")

liquido = salario - inss
```
